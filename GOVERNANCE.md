# Governance

## Scope

This document defines how `<org-name>` maintains and evolves this engineering guide.

## Roles

- Maintainers: approve major changes, manage releases, and enforce standards
- Contributors: propose and implement improvements
- Reviewers: provide technical and editorial feedback

## Decision model

- Prefer consensus through discussion in issues and pull requests
- If consensus cannot be reached, maintainers make a final decision
- Document significant decisions in pull request descriptions or changelog entries

## Change categories

- Editorial: wording, clarity, and formatting
- Guidance updates: process and technical recommendations
- Structural updates: file organization, templates, and automation

## Ownership

Use [CODEOWNERS](.github/CODEOWNERS) to define review ownership.

## Release cadence

Tag releases as needed when meaningful updates are bundled for downstream adopters.
