# Contributing

Thank you for contributing to this guide.

## Before you start

- Read the handbook files in [docs](docs)
- Check open issues to avoid duplicate work
- Open an issue for significant changes before implementation

## Contribution flow

1. Create a branch with a clear purpose
2. Make focused changes with clear commit messages
3. Run local checks:
   - `npm run lint`
   - `npm run format:check`
4. Open a pull request using the PR template
5. Respond to review feedback and keep discussion outcome-focused

## Pull request expectations

- Explain what changed and why
- Describe trade-offs and alternatives considered
- Include screenshots only if the change affects visual docs assets
- Keep scope manageable for reviewers

## Review principles

- Be kind, specific, and constructive
- Prioritize correctness, maintainability, and clarity
- Resolve feedback explicitly before merging

## Commit style

Use concise, imperative commit messages. Example:

- `docs: clarify ownership expectations in review section`
