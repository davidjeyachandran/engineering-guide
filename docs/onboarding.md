# Onboarding

## Goals

Help new team members become effective quickly while understanding system context and team norms.

## Week 1 checklist

- Set up local environment and required access
- Read core handbook chapters in this repository
- Shadow a code review and a planning discussion
- Complete a small starter contribution

## First 30 days

- Learn service boundaries and ownership model
- Understand release and incident processes
- Contribute a meaningful improvement with peer review

## Manager and mentor expectations

- Provide clear onboarding milestones
- Schedule regular check-ins for feedback and support
- Remove blockers quickly
